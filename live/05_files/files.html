1 Managing files in the operating system

1.1 rm

use v5.10;
unlink $file;
unlink @files;

1.2 mv

rename $old_name, $new_name;

The rename does not move files between file systems. For that, you can use: 

use File::Copy qw(move);
move $old_name, $new_name;

1.3 cp

There is no cp equivalent in core perl, so use File::Copy 

use File::Copy qw(copy);
copy $old_file, $new_file;

1.4 mkdir

Identical to the UNIX command

2 Reading and writing files

2.1 Shorthand for reading from a file

Special construct: (<>) reads from STDIN one line at a time. 

while (<>) {
    print if substr($_, 0, 7) eq '#+BEGIN'
}

cat files.org | grep_begin.pl

If you give your programme command line arguments, they are added to the program level default variable @ARGV. This while loop will treat them as filenames and pass all lines to $_; 

grep_begin.pl files.org

You can call this behaviour from command line, too, but it is usually done with regular expressions: 

perl -ne "print if /for/" *pl

FOr more perl command line options see: perldoc perlrun 

perl -ne "print if /^\#\+BEGIN/" files.org

2.2 Opening a file

See: 'perldoc -f open' and 'perldoc perlopentut'

A file handle is pointer to a file in a file system. It is held in a scalar variable.

You should always use the modern, three argument mode of open with autodie: 

use v5.10;
use strict; use warnings;
use autodie;

{ # code block to automatically close the filehandle in the end
    open my $F, "<", "input.txt";
    while (<$F>) {
        print;
    }
}

You can close the filehandle manually, too.

Without autodie, you'll have to test the success yourself and print the error message ($!): 

use strict; use warnings;
open my $fh, "<", "input.txt" or die "cannot open < input.txt: $!";
while (<>) {
    print;
}
close $f;

You can also test if the file exist before opening it: 

use v5.10;
use strict; use warnings;

my $file = 'input.txt';
say "File $file does not exist" and exit unless -e $file;
say "File $file is not readable" and exit unless -r $file;

open my $fh, "<", "input.txt" or die "cannot open < input.txt: $!";
while (<>) {
    print;
}
close $f;

Read more about file tests in 'perldoc -f -x'.

A complete example reading from a file, writing the results into a file and comments in a log: 

use v5.10;
use strict; use warnings;
use autodie;

my $file = shift; # read the first argument;
open my $IN,  "<",  $file;
open my $OUT, ">",  "$file.out";
open my $LOG, ">>", "$file.log";

while (<$IN>) {
    # modify the line and print it to $out
    chomp;
    say $OUT scalar reverse;    
}

say $LOG scalar localtime. "\tran programme $0";

Run the programme like this. 

open_multiple.pl grep_begin.pl

There is no printed output!

3 Running programmes from perl

Perl has backtics operator for running external programmes and capturing the output. Perl can be used instead of shell scripts to automate running of programmes.

Say you have created a programme: 

echo "perl -lne 'print scalar reverse'" > rev.sh;
chmod 755 rev.sh

And you want to run it on all .pl files in the directory: 

use v5.10;
@ls = split "\n", `ls *pl`;
say "files: ", "@ls";

for my $plfile (@ls) {
   say `rev.sh < $plfile`;
}


files: all_reverse.pl cds_in_embl.pl files.pl grep_begin.pl open_modern.pl open_multiple.pl open_older.pl

The above programme will plenty of output. Try it on command line!

There are many ways to open compressed files and stream their contents into a programme. This is the simplest that does not use any external modules and works in UNIX environments: 

use autodie;
my $file = 'seqs.fa.gz';
open my $FH, "zcat $file|";
while (<$FH>) {
  # do something
}

By default perl follows UNIX command line conventions to read files line by line. This is controlled by a global variable $/. It can be redefined to read multiple lines. 

{ # code block limits the extend of local declaration of a global variable
  local $/ = "\/\/\n";
  open my $SEQ, '<', shift;
  while (<$SEQ>) {
    my $seq = $_; 
    my ($ac) = $seq =~ /AC +(\w+)/;  
    say "$.: ", $ac if $seq =~ /FT +CDS/;
  }
}

For every line or block read, perl increases its internal counter $.
