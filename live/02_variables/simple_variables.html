1 Variables: simple values

1.1 What are variables

Variables are easy to spot: They all start with a special character. Simple variables are stored in scalar variables starting with the "$" character.

Perl is context sensitive language. One aspect of this is that there is only one variable type for strings, numbers and boolean values.

Be sure to understand the difference between variable and value!

The "=" is not "equals" but is the assignment operator. 

$a = 'a';
print $a;

a

String 'a' is stored in variable $a. 

$a = 2;
print $a;

2

1.2 Numbers

If we use a numerical operator on a value, the numerical value is extracted from a it. Numerical operators can be applied on string values. 

$a = '2a';
print $a + $a;

4

Certain rules apply, so do not try to misuse this feature!

Numbers have a few special formats in perl: 

$a = 2_000_000_000_000;
print $a * $a;

4e+24

1.3 Strings

Single-quoted strings are verbatim, double quotes allow interpolation of special characters and variables.

The special characters has to be escaped with a backslash "\" character. 

print "\tThis is a 'line' \\ \n";
print '\tThis is a \'line\' \\ \n';

	This is a 'line' \ 
\tThis is a 'line' \ \n

The quote-like operator makes it possible to write without escaping all the time. Any delimiter can be used – as long as it does not appear in the string. 

my $time = 5;
print qq("We'll finish at $time o'clock", I say.);

"We'll finish at 5 o'clock", I say.

1.6 Comparing values

Since scalar can hold both strings and numbers, comparison operators have to be different. 

print "Is a same as b? ", 'a' eq 'b', "\n";
print "Is a same as a? ", 'a' eq 'a', "\n";
print "Is 2 same as 3? ", 2 == 3, "\n";
print "Is 2 not same as 3? ", 2 != 3, "\n";
print "Is 2 same as 2? ", 2 == 2, "\n";

print "Is a smaller than as b? ", 'a' lt 'b', "\n";
print "Is 2 smaller than 3? ", 2 < 3, "\n";

1.9 Special versions of assigning

Operating and assigning at once: in-place operators 

$a = $a + $b;  # common need
$a += $b;      # a shorter and faster way
$a ||= 'def';  # assigning a default value

Multiple assignments 

$d = $c = $b = 2;
print "$d = $c = $b";
$b = 4 + ($a = 2);  # not recommended
# do the assignments separately
$a = 2;
$b = 4 + $a;

2 = 2 = 2

1.10 Scoping

All variable so far have been global, i.e. visible everywhere in the programme. Variables typically have different values in different parts of the programme. 

$state = 'outside';
{
    $state = 'inside';
    print "In : $state", "\n" 
}
print "Out: $state", "\n" 

In : inside
Out: inside

Lexical variables solve this problem by limiting the visibility to a block. This is done using keyword my before the variable. 

$state = 'outside';
{
    my $state = 'inside';
    print "In : $state", "\n" 
}
print "Out: $state", "\n" 

In : inside
Out: outside

Now you can see the benefits of limiting visibility of variables. The use strict pragma forces you to declare all variables. Using 'my' at top level makes the code available in all blocks under it – in effct global within the file. 

use strict;
my $state = 'outside'; # will not compile without 'my'
print "Out : $state", "\n";
{
    my $state = 'inside';
    print "In : $state", "\n"; 
}
print "Out: $state", "\n";

Out : outside
In : inside
Out: outside

1.12 Variable interpolation

Same rules apply as for special characters: Only double quoted strings are interpolated. 

my $aa = '1';
print "This is $aa\n";
print "This is ${aa}st\n"; # just like in BASH
$b = "${aa}st";
$b = $aa. "st";  # identical to above line

This is 1
This is 1st
