2 The perl programme structure

Computer programmes are composed of statements. In perl, every statement is ended by the semicolon ';' character. 

#!/usr/bin/env perl
print "Hello!\n";

Hello!

The execution is one statement at the time from the beginning of the file. Comments in perl are after a hash '#' character.

You should always use defensive programmings style. it means that you should protect your programmes from your own and other people's errors.

Best way to protect yourself from your own errors: Use the two standard pragmas (compiler instructions): 

use strict;
use warnings;
print "Hello\n";

Hello

Also, you can install a package (Modern::Perl) that does it (and other beneficial things, like defines the function say) for you. 

use Modern::Perl;
say "Hello";

Hello

The way how scripts are turned into executables depends on the operating system. Unix sets the execute permissions: 

perl hello.pl
chmod 755 hello.pl
./hello.pl
